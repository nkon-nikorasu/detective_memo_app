## このサービスへの思い・作りたい理由
  自分は趣味でミステリー小説をよく読みます。その際によく悩んでいたことがあって
  時系列や登場人物の情報が頭の中でうまく整理できなくなることです。
  ミステリー小説の醍醐味は探偵役の登場人物のように読者自体も推理することだと自分は思っているんですが
  ドラマや映画と違って小説はほとんど全て活字で物事を表現するので読者の想像力任せな部分が多いです。
  そこまで登場人物が多くなければ覚えることはあまり苦ではないのですが登場人物が多い物語や序盤に伏線と思える話題が出ても中盤終盤には忘れてしまったり事件の時系列や各登場人物の時系列が話題になると頭の中でごっちゃになってその後の物語をなんとなく楽しむ形になってしまいます。
  そこの負担をできるだけ取り除く方法があればいいと思ったのがきっかけです。

## ユーザー層について
  ・ミステリー小説を好んで読み謎解きが好きな20-40の方
  自分自身がこの層に該当しなおかつwebアプリの使用に抵抗がない年齢層だからです

## サービス利用イメージ
  ミステリ小説読書中に伏線、登場人物、時系列を任意に登録していく
    1.小説内の事件の登録
      事件名及び「殺人」、「密室」、「テロ」などのタグ登録
      読了済みか読書中のページ数登録
      時系列表の登録
      伏線メモの登録
    2.登場人物の登録
      「性別」、「年齢」、「職業」、「人物説明」などの登録
      時系列表による移動場所、行動の登録
      各登場人物の関係性の登録
    3.推理メモ
      任意の段階で読者がまとめた情報を整理した上での真相や犯人の予想の記述
  基本事件の登録を必須部分にしてそれ以外を任意で登録することで利用者が負担を感じずに利用できるように意識しています

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
